# Правила перевода

## Общие принципы

- Перевод должен быть **точным**, но **читабельным** — не дословным там, где это мешает пониманию.
- Сохраняйте структуру оригинала: заголовки, списки, таблицы, блоки кода.
- Код и имена переменных/функций **не переводятся**.
- Технические термины, не имеющие устоявшегося русского аналога, оставляются на английском или вносятся в [глоссарий](./glossary.md).

## Глоссарий

Перед переводом нового термина — проверьте [glossary.md](./glossary.md). Если термина нет, добавьте его туда с пояснением.

## Структура файлов глав

Каждая глава хранится в отдельной папке:

```
chapters/
└── 01-chapter-name/
    └── translation.md
```

Имя папки: `NN-slug`, где `NN` — двузначный номер главы, `slug` — краткое название главы строчными буквами через дефис (латиница).

## Оформление файла перевода

Каждый файл `translation.md` начинается с шапки:

```markdown
---
chapter: 1
title: "Название главы"
original_title: "Original Chapter Title"
status: in-progress # in-progress | review | done
translator: Имя
---
```

## Статусы перевода

| Статус        | Описание                         |
| ------------- | -------------------------------- |
| `in-progress` | Перевод в процессе               |
| `review`      | Перевод завершён, нужна проверка |
| `done`        | Проверено и готово               |

## Форматирование

- Используйте **жирный** для выделения ключевых понятий при первом упоминании.
- Сноски и примечания переводчика оформляйте как: `> **Примечание переводчика:** текст`.
- Блоки кода сохраняйте без изменений, при необходимости добавляйте комментарий на русском.
